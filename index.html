How it works

Install once. Route forever.

STEP 1

Start the router

Run NadirClaw locally. It sits between your app and OpenAI's API. No cloud services, no signup, no tracking.

STEP 2

Point your tools to localhost

Change your base URL from api.openai.com to localhost:8856. Works with Claude Code, Cursor, Codex, Aider, or any OpenAI-compatible client.

STEP 3

Watch costs drop

NadirClaw classifies every prompt and routes it to the cheapest model that can handle it. Check the dashboard to see where you're actually spending.

Why not just use rules?

The most common question: "Can't I just write if/else logic instead?"

Manual rules

Breaks on edge cases

Rules like "short prompt = cheap model" fail immediately. "Fix the auth bug" is short but complex. "Explain the entire codebase architecture in detail" is long but a simple retrieval. You end up maintaining a growing regex graveyard.

Gateway + heuristics

Works, but you build everything

LiteLLM gives you multi-provider access. But you still write the routing logic, the classifier, the fallback chains, the cost tracking, the dashboard. NadirClaw ships all of that. pip install and done.

Safe mode transforms

  • JSON minification — compact pretty-printed JSON without changing values
  • Tool schema dedup — repeated tool schemas replaced with short references
  • System prompt dedup — duplicated system text removed from later messages
  • Whitespace normalization — collapse runs of blanks, skip code blocks
  • Chat history trimming — keep system + first turn + last N turns
# enable on your server
nadirclaw serve --optimize safe

# or per-request
{"optimize": "safe", "model": "auto", ...}

# dry-run on any file
nadirclaw optimize payload.json

All transforms are lossless. Code blocks, URLs, and unicode are never modified. Off by default — zero overhead when disabled.

Frequently asked questions

Everything you need to know about NadirClaw.

What is NadirClaw?

NadirClaw is an open-source LLM router that sits between your application and LLM providers. It analyzes prompt complexity in real-time and automatically routes simple requests to cheaper models while keeping complex tasks on premium models, saving 40-70% on API costs without changing any code.

How does NadirClaw reduce costs?

NadirClaw classifies each prompt by complexity. Simple tasks like status checks, formatting, and basic Q&A are routed to cheaper models (Claude Haiku, GPT-4o-mini). Complex tasks like code generation and reasoning stay on premium models (Claude Opus, GPT-4). You pay less per token on average.

Does NadirClaw require code changes?

No. NadirClaw is a drop-in proxy. Point your LLM client's base URL to NadirClaw instead of the provider directly. It works with Claude Code, Cursor, Aider, and any OpenAI-compatible client.

Is NadirClaw free?

Yes. NadirClaw is free and open-source under the MIT license. You can self-host it on your own infrastructure at no cost.

What LLM providers does NadirClaw support?

NadirClaw supports Anthropic (Claude), OpenAI (GPT), and Google (Gemini) models out of the box. You can configure custom model tiers and routing rules for any OpenAI-compatible provider.

How do I install NadirClaw?

Run pip install nadirclaw or use Docker: docker run -p 8000:8000 nadirclaw/nadirclaw. Configuration is done through environment variables or a config file.

How does the classifier work?

NadirClaw uses sentence embeddings (DistilBERT-based) to compute cosine similarity against pre-trained centroid vectors for simple and complex prompts. Classification takes ~10ms. It also detects agentic workflows (tool calls), reasoning chains ("step by step"), and vision content (images) as separate routing signals.

What happens when the classifier is wrong?

NadirClaw biases toward the complex model on low-confidence classifications (threshold: 0.06). This means it's more likely to over-serve (send a simple prompt to a premium model) than under-serve. You can test any prompt with nadirclaw classify "your prompt" and tune the threshold via NADIRCLAW_CONFIDENCE_THRESHOLD.

How much latency does routing add?

~10ms for the sentence embedding classification. The router runs locally on your machine, so there is no network hop. For comparison, a typical LLM API call takes 500ms-5s. The routing overhead is less than 1% of total request time.

Where does the "40-70% savings" claim come from?

In typical coding sessions, 60-70% of prompts are simple (file reads, formatting, basic questions, docstrings). If those route to models that cost 10-20x less per token, your blended cost drops 40-70%. Your actual savings depend on your prompt mix. Run nadirclaw report to see your real breakdown after a day of use.
